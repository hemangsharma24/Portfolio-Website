Design System & Style Guide · v1.0

A warm, editorial system built for case-study storytelling.

This guide defines the visual language for hemangsharma.com — a portfolio whose job is to make a recruiter understand who you are, see proof you ship outcomes, and want to read a case study, all within the first scroll. Every token below is rendered live in the system it documents.

01 · Design Direction

Warm Editorial Minimal

Not Modern-SaaS (too cold for your voice), not maximalist (fights scanning). The direction is editorial — like a well-set design magazine: a confident serif for statements, a clean grotesque for everything functional, generous whitespace, and restraint with color so the work is the loudest thing on the page. It suits a designer who also writes reflections and poetry, and it reads as senior and considered to a hiring manager.

Brand personality

Considered, warm, candid, outcome-driven. Confident without shouting. The burgundy gives it gravity and warmth; the cream keeps it human rather than clinical.

Guiding principles

01

Outcomes over process

Every project leads with a result a recruiter can grasp in one glance, not a paragraph about method.

02

Let the work be the color

Surfaces stay neutral. Product screenshots supply the vibrancy. One accent, used sparingly.

03

Scan first, read second

Title → tag → metric → CTA must communicate before any body copy is read.

04

One system, repeated

Cards, buttons, and spacing repeat identically. Consistency itself signals systems thinking.

02 · Color System

Anchored on burgundy & cream

Your existing nav already establishes burgundy on cream — this system formalizes it into a ramp and adds exactly one accent. White text on the burgundy primary measures ~13.8:1; primary ink on paper measures well past AAA. Color encodes meaning, never decoration.

03 · Typography

Fraunces + Hanken Grotesk

Fraunces (a warm, optical-sized serif) carries the hero and section headings — it gives the editorial character and matches your reflective voice without sacrificing readability. Hanken Grotesk handles all UI, labels, and body: friendly, highly legible, and not the over-used Inter. Both are free on Google Fonts. The scale below follows a ~1.25 ratio.

04 · CTA System

Four button tiers, one verb each

Primary — the one action per view

View case study ↗ Let's work together

bg #5C2435 · text #FBF6F0 · padding 12/22 · radius 12 · 15px/600 · hover → #4A1C2A + −1px lift · focus ring 3px rgba(92,36,53,.35)

Secondary — supporting action

Download résumé See all projects

transparent · 1px #3A1620/24 border · text #5C2435 · hover → fill #F6EEF1 + border #5C2435

Ghost — low-emphasis / nav-level

Contact me Back to top

no border/bg · text #5C2435 · padding 10/14 · hover → fill #F6EEF1

Text link — inline & card CTAs

Read case study → View on Behance ↗

text #5C2435 · 600 · hover → color #8F3C1E + 1.5px underline #C2562F — use this for the per-card CTA so cards stay light.

Hierarchy rule: exactly one Primary per screen section. Hero = Primary "View work" + Secondary "Download résumé". Each project card = a single Text-link CTA. The contact section = Primary "Let's work together". Never stack two primaries.

05 · Layout & Spacing

8-point system, 1180px container

  • Container: max-width 1180px, side gutters 24px desktop / 16px mobile.
  • Grid: 12 columns, 24px gutter. Hero splits 7/5 (statement / tags). Project cards are full-bleed within the container, content half + thumbnail half (≈58/42).
  • Section spacing: 96px vertical padding desktop, 56px mobile, with a 1px hairline divider between major sections.
  • Card padding: 32px desktop, 20px mobile.
  • Component gaps: 12px within a group (tags, button rows), 24px between groups.

06 · Featured Project Cards

One template, four projects

Light surface, single muted top-accent, and a fixed content order so the four cards read as one system. Max description length: one line, ≤140 characters. The metric chip is the most important element — it carries the outcome above the fold.

Content structure (top → bottom)

  • Index — 01 muted, for rhythm.
  • 2–3 tags — domain · type · scope. Pill style, never more than three.
  • Title — product name + what it is, one line.
  • Description — one line, ≤140 chars, the "what + for whom".
  • Metric chip — one quantified outcome. The single most scannable element.
  • Role — "Sole designer" / "Founding designer" — answers a recruiter's first question.
  • CTA — text-link "Read case study →".
  • Thumbnail — fills the opposite 42%; alternate sides per card for rhythm.

What to cut: the current 3–4 line paragraphs. Move that narrative into the case study itself, where it belongs. The card's only job is to earn the click.

07 · Navigation & Header

Quiet, sticky, one accent cue

  • Style: sticky burgundy bar (already your instinct), full name as wordmark — drop the vowel-stripped variant; clarity beats cleverness for recruiters.
  • Active state: a 2px terracotta underline under the current page (shown in this guide's own header) — the single place the accent appears in the nav.
  • Hover: inactive links lighten from Burgundy-300 to full cream over 180ms.
  • Resume: keep it visually distinct (brighter) since it's a common recruiter target, but as a Secondary-weight cue, not a second Primary.
  • Mobile: collapse to a hamburger → full-height sheet, links at 20px with 16px vertical rhythm, Primary "Let's work together" pinned at the bottom of the sheet.
08 · Section-by-Section

Applying the system to each block

Hero

Add a credibility line (years + "live with a US insurer") and a 3-up outcomes band. Demote to one Primary + one Secondary CTA. Replace the pixel avatar with an "HS" monogram or headshot; keep the pixel art for the personal footer.

Featured Projects

Adopt the single light-card template above. The screenshots become the only saturated color on the page.

Work Experience

Keep the clean three-column rows but reconcile dates with the résumé (OverseeAI ends Jun 2026, Divum ends Nov 2024). Add a thin hairline and a muted section background (--surface-2) so it reads as a distinct "résumé" block.

Company logo wall

Move it directly under the hero — these are recognizable brands and belong where credibility is built. Fix the broken {param} and text-only "Dyson" tiles; normalize all logos to a single grayscale tone at equal optical size so none dominates.

Testimonials

Strong already. Replace the four tinted card backgrounds with the neutral surface + a small accent quote-mark, for consistency with the new card system. Two columns desktop, one mobile.

Locked / confidential work

Turn the dead-end into a soft CTA: "Happy to walk through these confidentially → Get in touch."

Personal + contact footer

Keep the warmth. Make the email a Primary "Let's work together" action; keep copy-to-clipboard as a secondary affordance. Fix the ~ -^ sign-off glyph.

09 · Accessibility & Readability

AA as the floor, AAA where easy

Built in

All text/background pairs in this guide clear WCAG AA (most clear AAA). Primary ink ~14:1, secondary ~7:1, muted ~5:1, metric chip ~4.6:1, white-on-burgundy ~13.8:1.

Watch for

Don't drop muted text below 14px on paper. Don't put body copy in the terracotta accent. Don't rely on the category color alone — always pair it with the text tag.

  • Focus: visible 3px focus ring on every interactive element (shown on the buttons above).
  • Targets: minimum 44×44px tap area on mobile for nav and CTAs.
  • Motion: all entrance animation wrapped in prefers-reduced-motion.
  • Cognitive load: one-line card descriptions, one Primary per section, max 3 tags — all reduce what the eye must process.
  • Line length: body capped at ~70ch for comfortable reading.
